# MedVis3D Outreach Operating Plan

## Outreach Thesis

Manual founder-led only until the first proof pack passes route, privacy, artifact, and
browser-smoke gates. The first marketable lane is not hospital IT integration. It is
patient-specific visual explanation for clinics and imaging groups that already need to
explain 2D images to patients.

Core message:

MedVis3D turns ordinary 2D medical images into confidence-labeled interactive 3D
educational consult views in the browser. It is educational visualization, not diagnostic
output.

## Buyer Lanes

| Rank | Lane | Why now | First proof pack |
|---:|---|---|---|
| 1 | Outpatient imaging centers and radiology groups | Patient-facing report/image access creates explanation burden | patient_specific_3d_consult_builder |
| 2 | Orthopedic, sports medicine, dental, spine clinics | Clinicians already explain images during visits | xray_explanation_pack |
| 3 | Hospital patient engagement and digital health teams | Need safer patient-friendly context around imaging | patient_portal_explanation_pack |
| 4 | Imaging informatics and PACS leaders | Need standards credibility, but longer sales cycle | dicom_trust_pack |
| 5 | Medical education and simulation programs | Can buy educational visuals without clinical claims | education_case_library_pack |
| 6 | Healthcare innovation labs | Good feedback channel, not first revenue assumption | patient_specific_3d_consult_builder |

## Manual Rules

- Manual founder-led only.
- Maximum 3 new LinkedIn sends per weekday.
- Maximum 2 contacts per organization.
- Use one buyer lane, one proof pack, one ask, and one caveat per message.
- Always include the caveat: educational visualization, not diagnostic output.
- Do not automate sending until a repeatable reply pattern exists.
- Stop outreach for any lane where prospects interpret the product as diagnostic AI.

## LinkedIn Opener

Hi {{first_name}}, I am building MedVis3D, a browser-local 2D-to-3D consult builder
that helps clinicians explain ordinary medical images to patients using an interactive
educational 3D view. It is educational visualization, not diagnostic output.

Would you be open to a 20-minute review of a de-identified demo workflow for
{{buyer_lane}}?

## Follow-Up

Thanks again for considering this. The narrow workflow I am testing is:
upload or capture a de-identified image, enhance it locally, generate a confidence-labeled
3D educational view, add a callout, and export a consult snapshot. No diagnostic findings
or treatment recommendations are generated.

If this kind of patient explanation artifact is useful in your setting, I would value
20 minutes of feedback.

## Logging Fields

Track every send manually:

- prospect_name
- role
- organization
- buyer_lane
- proof_pack_id
- route_or_demo_asset
- claim_boundary
- caveat_used
- message_variant
- sent_date
- reply_status
- objection
- next_action

## Stop Conditions

Stop or rewrite outreach if any message implies:

- FDA-cleared software.
- HIPAA-compliant secure sharing.
- Diagnostic interpretation.
- PACS replacement.
- Surgical planning.
- Disease detection.
- Guaranteed true 3D anatomy from unclear 2D images.

## First 14-Day Outreach Run

Run only after `patient_specific_3d_consult_builder` passes the proof gates:

1. Prepare 10 highly relevant prospects across imaging centers and specialty clinics.
2. Send 3 messages per weekday maximum.
3. Offer a 20-minute review, not a purchase ask.
4. Record objections verbatim.
5. Convert only warm replies into a 14-day pilot discussion.
6. Update the proof pack before scaling message volume.
